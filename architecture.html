🤖 LLM Orchestration

Projects for serving open-source LLMs at low cost on Azure Spot infrastructure, with automatic failover and OpenAI-compatible APIs.

⚡
az-spot-orchestrator
Dynamic Spot VM LLM Control Plane
Finds the cheapest Azure region, provisions a Spot VM with Ollama via cloud-init, and proxies inference traffic to it. On eviction, Temporal automatically re-provisions in a new region.
Python Terraform Docker Azure Spot Ollama FastAPI
You ──OpenAI-compatible──▶ Control Plane VM FastAPI :8000 + Temporal Cosmos DB (serverless) │ provisions ▼ Spot VM (cheapest region) Ollama :11434 + free temp disk

az-llm-aks
LLM Orchestrator on AKS
Kubernetes-native LLM orchestrator on Azure AKS. Control plane handles routing; KEDA scales Spot node pools for inference workloads. ELK stack for observability.
Python Terraform AKS / Kubernetes Azure Spot KEDA Docker
Control Plane (LoadBalancer :80) └── routes to Spot Node Pool └── Ollama workers (KEDA-scaled) Observability: Fluent Bit → Elasticsearch → Kibana

az-llm-proxy
LiteLLM Proxy for Azure AI Foundry
Single OpenAI-compatible endpoint on :4003 that fans out to Azure-hosted models (DeepSeek, Phi-4). Includes a custom hook that transparently translates tool calls for Phi models which lack native tool support.
Python Docker Compose Azure AI Foundry LiteLLM
:4003 (OpenAI-compatible) └── LiteLLM router ├── deepseek-v3.2-speciale ├── phi-4-reasoning ├── phi-4 └── phi-4-multimodal-instruct PhiTextToolCalling hook: text→tool call translation
🌐
vertex-proxy
OpenAI Proxy for Google Vertex AI
LiteLLM-based proxy on :4001 that exposes Google Vertex AI Gemini models via the OpenAI API format. Drop-in replacement for any OpenAI SDK client targeting Gemini.
Python Docker GCP Vertex AI LiteLLM
:4001 (OpenAI-compatible) └── LiteLLM ├── gemini-2.5-flash ├── gemini-2.5-pro └── gemini-3.1-flash-lite-preview Auth via GCP Service Account JSON

llm-orchestrator
Single-CLI Ephemeral Spot LLM
One Python CLI that queries Azure Retail Prices API, spins up a Spot VM, runs Ansible for setup, deploys SmolLM2-360M via llama-server, and returns an OpenAI-compatible endpoint — all in one command.
Python Ansible Azure Spot llama-server
main.py 1. Price API → cheapest region 2. Deploy Standard_B2ts_v2 Spot VM 3. Ansible: base setup + NetBird VPN 4. Deploy SmolLM2-360M-Instruct Q4_K_M 5. Expose :8080 (OpenAI-compatible, API key)

Personal Finance & Document Intelligence Pipeline
Ingests invoices/receipts from Gmail, extracts structured data + embeddings via GPT-4o/Gemini, and stores them for hybrid RAG querying (Text-to-SQL + vector search). Supports historical backfill and real-time Gmail Push Notifications via GCP Pub/Sub.
